Arbetsprocess

Här beksrivs arbetsprocessen för utveckling och publicering av webbplatsen.

Webbprojektet finns tillgängligt i ett publikt repo på GitHub: https://github.com/Safe-Pizza/DT211G_Lab1_Vite.git

Utveckling

GitHub repot klonas till din lokala dator. NodeJS används för att automatisera uppgifter, paketen finns i en package.json som återställs via kommandot nmp install. All utveckling av webbplatsen sker i branchen dev där du kan arbeta med källkodsfilerna. HTML-filer läggs i rot-katalogen, JavaScript-filer och CSS läggs i src-katalogen och filer som inte ska processas läggs i public-katalogen. För skapa en demowebbplatsen under utvecklingen använder du kommandot npm run dev, då får du en länk där du kan se webbplatsen lokalt i din webbläsare. När du sedan vill testköra webbplatsen innan publicering använder du kommandot npm run build, då byggs webbplatsen och en katalog för distribution skapas (dist) där optimerade och minifierade publiceringsfiler skapas, och sedan kommandot npm run preview för att få en lokal länk till den färdiga webbplatsen. Fungerar webbplatsen som tänk är du redo att gå vidare med publicering. Dist-katalogen som skapades när kommandot npm run build gjordes raderas innan publicering. Hur du går vidare med publicering finns beskrivet under rubriken publicering.

Om inte så raderar du dist-katalogen och fortsätter utveckligen i dev-branchen för att sedan köra kommando för build och preview igen för testning.

Publicering

För att publicera webbplatsen så görs en merge med main-branchen, hantera då eventuella konflikter. När en merge är gjord kommer webbplatsen automatiskt byggas och uppdateras.

Automatisering

Webbprojektet innehåller automatiserade uppgifter.

Vite

Vite processar källkodsfilerna som minifieras och optimeras för publicering samt skapar en katalog för publicering (dist).

Vite Image Optimizer

Vite Image Optimizer komprimerar bilder för publicering. Filformat som komprimeras är: png, jpg, webp och avif.

Publicering via Netlify

Netlify är kopplat till git-repot main på GitHub som automatiskt kör byggkommando och publicerar webbplatsen.
